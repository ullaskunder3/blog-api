Imagine you're playing Minesweeper вҖ” you click a cell, and suddenly an entire row and column light up. That's the visual for this problem. When a `0` is found, its whole row and column must become `0`. Sounds simple, but the **real challenge** is doing it without extra space. рҹ’Ҙ

## рҹ§© The Problem

> Given an `m Г— n` integer matrix, if an element is `0`, set its **entire row and column** to `0`s.
>
> You must do it **in place**.

Two `0`s at `(0,0)` and `(0,3)` вҖ” row 0 is fully zeroed, columns 0 and 3 are fully zeroed.

### Constraints
- `m == matrix.length`
- `n == matrix[0].length`
- `1 <= m, n <= 200`
- `-2ВіВ№ <= matrix[i][j] <= 2ВіВ№ - 1`

---

## Step 1 вҖ” Start with the obvious solution

If you see a `0` at `(i, j)`:
- **row i** must become all `0`s
- **column j** must become all `0`s

So the most natural idea is:

```python
rows = set()
cols = set()
```

**First pass** вҖ” scan and record:
```
whenever matrix[i][j] == 0:
    add i to rows
    add j to cols
```

**Second pass** вҖ” apply:
```
if row in rows OR col in cols:
    make it 0
```

```python
class Solution(object):
    def setZeroes(self, matrix):
        m, n = len(matrix), len(matrix[0])
        rows = set()
        cols = set()

        for i in range(m):
            for j in range(n):
                if matrix[i][j] == 0:
                    rows.add(i)
                    cols.add(j)

        for i in range(m):
            for j in range(n):
                if i in rows or j in cols:
                    matrix[i][j] = 0
```

That gives:
- **Time:** O(m Г— n)
- **Space:** O(m + n) вҖ” for the two sets

This is already good. вң…

---

## Step 2 вҖ” Interview follow-up: can we do O(1) space?

Now comes the trick.

Instead of extra arrays/setsвҖҰ **use the matrix itself as storage.**

### рҹ’Ў The Core Insight

The **first row** can store вҶ’ *which columns should become zero*

The **first column** can store вҶ’ *which rows should become zero*

Let's see it with Example 1:

```
Original:          After marking:
в”Ңв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”¬в”Җв”Җв”Җв”җ     в”Ңв”Җв”Җв”Җв”¬в”Җ0в”Җв”¬в”Җв”Җв”Җв”җ   вҶҗ first row stores column markers
в”Ӯ 1 в”Ӯ 1 в”Ӯ 1 в”Ӯ     в”Ӯ 1 в”Ӯ 0 в”Ӯ 1 в”Ӯ
в”ңв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Ө     в”ңв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Ө
в”Ӯ 1 в”Ӯ 0 в”Ӯ 1 в”Ӯ     в”Ӯ 0 в”Ӯ 0 в”Ӯ 1 в”Ӯ   вҶҗ first col stores row markers
в”ңв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Ө     в”ңв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Ө
в”Ӯ 1 в”Ӯ 1 в”Ӯ 1 в”Ӯ     в”Ӯ 1 в”Ӯ 1 в”Ӯ 1 в”Ӯ
в””в”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”ҳ     в””в”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”ҳ
```

When we find `matrix[1][1] == 0`:
- Mark **row 1**: `matrix[1][0] = 0`
- Mark **column 1**: `matrix[0][1] = 0`

Now the first row/column act like markers. Smart, right?

### вҡ пёҸ But there's a problemвҖҰ

What if the **first row itself** originally had a zero? Or the **first column**?

Because we're repurposing them as marker storage, we would **lose that information**.

So we separately store:
```python
firstRowZero = False
firstColZero = False
```

Two booleans. That's it. Still O(1) space. рҹҺҜ

---

## рҹ§  Full Thinking Process

### Pass 1 вҖ” Protect the first row & column

Determine:
- Should the first **row** become zero?
- Should the first **column** become zero?

### Pass 2 вҖ” Mark using inner cells

For every **inner cell** `(i, j)` where `i >= 1` and `j >= 1`:

If it's zero вҶ’ mark its row and column:
```
matrix[i][0] = 0    вҶҗ mark the row
matrix[0][j] = 0    вҶҗ mark the column
```

### Pass 3 вҖ” Zero inner cells using markers

If `matrix[i][0] == 0` OR `matrix[0][j] == 0` вҶ’ zero it out:
```
matrix[i][j] = 0
```

### Pass 4 вҖ” Handle first row/column last

Apply the saved booleans to zero them out **if needed**.

> вҡ пёҸ **Order matters!** We must handle the first row/column **last**, otherwise we'd corrupt the markers before using them.

---

## рҹ’» Final Solution (O(1) Space)

```python
class Solution(object):
    def setZeroes(self, matrix):
        """
        :type matrix: List[List[int]]
        :rtype: None Do not return anything, modify matrix in-place instead.
        """
        m = len(matrix)
        n = len(matrix[0])

        firstRowZero = False
        firstColZero = False

        # Pass 1: Check if first row has any zero
        for j in range(n):
            if matrix[0][j] == 0:
                firstRowZero = True

        # Pass 1: Check if first column has any zero
        for i in range(m):
            if matrix[i][0] == 0:
                firstColZero = True

        # Pass 2: Use first row/column as markers
        for i in range(1, m):
            for j in range(1, n):
                if matrix[i][j] == 0:
                    matrix[i][0] = 0
                    matrix[0][j] = 0

        # Pass 3: Zero inner cells based on markers
        for i in range(1, m):
            for j in range(1, n):
                if matrix[i][0] == 0 or matrix[0][j] == 0:
                    matrix[i][j] = 0

        # Pass 4: Zero first row if needed
        if firstRowZero:
            for j in range(n):
                matrix[0][j] = 0

        # Pass 4: Zero first column if needed
        if firstColZero:
            for i in range(m):
                matrix[i][0] = 0
```

## вҸұпёҸ Complexity Analysis

| Approach | Time | Space |
| :--- | :--- | :--- |
| **Naive (extra sets)** | O(m Г— n) | O(m + n) |
| **Optimal (in-place markers)** | O(m Г— n) | **O(1)** вң… |

Both approaches scan every cell at least once (unavoidable), so time is always O(m Г— n). The difference is purely in space.

---

## рҹ“Ҡ Comparison: O(m+n) vs O(1) Space

| Feature | Set-based (O(m+n)) | In-place markers (O(1)) |
| :--- | :--- | :--- |
| **Space** | O(m + n) | O(1) вҖ” just 2 booleans |
| **Code simplicity** | Straightforward | Needs careful ordering |
| **Edge cases** | Simple | First row/col need special handling |
| **Interview impression** | Good starting point | Shows deep thinking рҹ’Ў |

---

## вҡ пёҸ Common Mistakes

1. **Zeroing the first row/column too early** вҖ” If you handle them in Pass 3 instead of Pass 4, you'll corrupt the markers and get wrong answers. The order is critical.

2. **Forgetting to scan the first row/column separately** вҖ” The inner loop starts at `(1,1)`, so the first row and column aren't covered by the marking phase. That's why we need `firstRowZero` and `firstColZero`.

3. **Modifying while scanning** вҖ” If you zero cells during the same pass you're scanning for zeros, you'll create "cascading zeros" that weren't in the original matrix. Always separate the scan phase from the modify phase.

4. **Starting inner loops at 0 instead of 1** вҖ” The inner marking and zeroing loops must start at index `1`, not `0`. Starting at `0` would overwrite the markers themselves.

## рҹ§  How to Learn to Think Like This

For matrix problems, ask yourself:

### 1. What information do I need later?

Here:
- Which **rows** need to be killed
- Which **columns** need to be killed

### 2. Can existing input store that info?

This is the big interview trick вҖ” **reuse the matrix itself**.

Common in:
- Matrix problems
- Linked list problems (e.g., marking visited nodes)
- Array problems (e.g., using negative values as flags)

### 3. What special cases break the trick?

Here:
- The **first row** вҖ” because we're using it as column storage
- The **first column** вҖ” because we're using it as row storage

So we protect them with two booleans.

---

## рҹ”‘ Pattern to Remember

> **"Use first row/column as state storage"**

This is a classic in-place trick. You'll see variations of it in:
- This problem (Set Matrix Zeroes)
- Game of Life (LeetCode 289)
- Any problem asking "modify in-place with O(1) extra space"

The general pattern:
1. **Protect** вҖ” save special-case info separately
2. **Mark** вҖ” use existing structure as storage
3. **Apply** вҖ” use marks to make changes
4. **Restore** вҖ” handle the special cases last

## рҹҺҜ Interview Tips

1. **Start with the O(m+n) solution** вҖ” Show you understand the problem first. Use sets to track rows and columns. It's clean and correct.

2. **Then optimize** вҖ” When the interviewer asks "can you do better on space?", pivot to the in-place marker approach. This is where you shine.

3. **Explain the ordering** вҖ” The most common follow-up is "why do you handle the first row/column last?" Be ready to explain that doing it early would corrupt your markers.

4. **Draw it out** вҖ” This problem is much easier to explain with a visual. Sketch the matrix on the whiteboard and show the marking step.

5. **Know the trade-off** вҖ” Both solutions are O(mГ—n) time. The only difference is space. Make sure to articulate *why* the in-place approach is better in memory-constrained environments.
